## 1. Project goal

Benchmark five frontier large-language-model APIs on Bayesian and inferential statistical reasoning, with an out-of-family LLM judge that scores assumption-compliance and reasoning quality alongside numerical accuracy. The headline contribution is the **three-ranking framework** — accuracy, robustness, and calibration — and empirical evidence that the rank order of the same five models depends on which axis you measure.

---

## 2. Research questions

The original RQ frame is preserved; obsolete sub-claims are flagged.

- **RQ1.** How accurately do frontier LLMs solve closed-form Bayesian problems (Phase 1 conjugate models)?
- **RQ2.** How accurately do they solve computational Bayesian methods (Phase 2: Gibbs / MH / HMC / RJMCMC / VB / ABC / hierarchical)?
- **RQ3.** Where does each model fail — what is the dominant L1 failure category (assumption violation, mathematical error, formatting, conceptual)?
- **RQ4.** How robust are model accuracies under three perturbation axes (rephrase / numerical / semantic)?
  - *Note:* the original RQ4 was scoped as "75 perturbations on 25 base tasks" using `data/raw_data/synthetic/perturbations.json` and `data/processed_data/results_v1/rq4_analysis.json`. The current canonical scope is **473 perturbations covering all 171 base tasks** (`data/raw_data/synthetic/perturbations_all.json`), with results in `data/processed_data/results_v2/{robustness_v2.json, perturbation_runs.jsonl}`. Original RQ4 retained as a legacy baseline; the v2 scope subsumes it.
- **RQ5.** How well-calibrated is each model's stated confidence vs its empirical accuracy? (ECE, see `data/processed_data/results_v2/calibration.json` and per-dim variant in `per_dim_calibration.json`.)

## 4. Task suite

171 base tasks + 473 perturbations across 38 distinct task types. Generators are deterministic and reproducible — task JSON files are **never edited manually**, only regenerated from source.

### Phase 1 — closed-form conjugate models (136 tasks, 31 task types)

Generated by [code/data_preprocessing/bayesian/build_tasks_bayesian.py](code/data_preprocessing/bayesian/build_tasks_bayesian.py) → [data/raw_data/benchmark_v1/tasks.json](data/raw_data/benchmark_v1/tasks.json). Covers Beta–Binomial, Gamma–Poisson, Normal-mean (known/unknown σ²), Normal–Inverse-Gamma, Dirichlet–Multinomial, Bayesian linear regression, decision theory (Bayes risk, posterior median/mean/mode under loss), prior sensitivity / improper priors, and stress tests (Box–Muller, inverse transform).

### Phase 2 — computational Bayesian methods (35 tasks, 7 task types)

Generated by [code/data_preprocessing/bayesian/build_tasks_advanced.py](code/data_preprocessing/bayesian/build_tasks_advanced.py) → [data/raw_data/benchmark_v1/tasks_advanced.json](data/raw_data/benchmark_v1/tasks_advanced.json). Five tasks per type, with `np.random.seed(42)` for ground-truth MC estimates.

| Type | Method | Tolerance |
|---|---|---|
| GIBBS | Gibbs Sampling | 0.05 |
| MH | Metropolis–Hastings | 0.05 |
| HMC | Hamiltonian Monte Carlo | 0.05 |
| RJMCMC | Reversible Jump MCMC | 0.05 |
| HIERARCHICAL | Hierarchical Bayes | 0.05 |
| VB | Variational Bayes | 0.10 |
| ABC | Approximate Bayesian Computation | 0.10 |

Combined task set: [data/raw_data/benchmark_v1/tasks_all.json](data/raw_data/benchmark_v1/tasks_all.json) (171 records, 38 distinct task-type prefixes).

### Perturbations — three axes, 473 records

## 5. Scoring methodology

Three scoring paths. The keyword-rubric paths (A and B) must remain weight-aligned; the external judge is independent.

### NMACR per-task components — five dimensions in [0, 1]

| Symbol | Component | Weight | Source |
|---|---|---:|---|
| N | Numerical Accuracy | 0.10 | extracted numbers vs ground truth |
| M | Method / Structure | 0.20 | required reasoning steps present |
| A | Assumption Compliance | 0.30 | required statistical assumptions stated |
| C | Confidence Calibration | 0.15 | confident-when-correct / uncertain-when-wrong |
| R | Reasoning Quality | 0.25 | step-by-step derivation + interpretation |

Weights are literature-derived (sole canonical scheme since Approach A, 2026-05-03). Defended in [code/analysis/metrics.py](code/analysis/metrics.py) docstring; full literature trail in `llm-stats-vault/90-archive/audit/methodology_continuity.md` §"NMACR weighting" (Du 2025, Boye & Moell 2025, Yamauchi 2025, ReasonBench 2025, Wei 2022, Chen 2022, Bishop 2006, Nagarkar 2026, FermiEval 2025, Liu 2025).

> **Note:** [CLAUDE.md](CLAUDE.md) §"Scoring" still states `N=M=A=C=R=0.20` (equal-weight). The actual implementation in both [code/models/response_parser.py](code/models/response_parser.py#L29) and [code/analysis/metrics.py](code/analysis/metrics.py) uses the literature-derived NMACR weights above. CLAUDE.md is stale on this point. [TODO: confirm whether to update CLAUDE.md or revert metrics.py.]

### Path A — live runner (deterministic keyword rubric)

[code/models/response_parser.py](code/models/response_parser.py): `full_score(raw_response, task) -> dict`. Used by `code/models/run_all_tasks.py` to score each model response as it streams in. CONCEPTUAL tasks scored as `1.0 × rubric_score`; tasks with both numeric targets and rubric structure scored as `0.6 × numeric + 0.4 × rubric`. Pass threshold `final_score ≥ 0.5`.

### Path B — post-hoc (TaskRun dataclass aggregation)

[code/analysis/metrics.py](code/analysis/metrics.py): `score_all_models(tasks, runs)` operates on `TaskRun` dataclasses parsed from `runs.jsonl`. Same NMACR weights, same numeric/rubric blend. Tier-5 stress-test multiplier 1.50 (no Tier-5 tasks currently exist).

### External judge — Llama 3.3 70B Turbo via Together AI

[code/analysis/llm_judge_rubric.py](code/analysis/llm_judge_rubric.py): independent re-scoring of each `(task, response)` pair on four dimensions: `METHOD_STRUCTURE`, `ASSUMPTION_COMPLIANCE`, `REASONING_QUALITY`, `REASONING_COMPLETENESS`. Model is `meta-llama/Llama-3.3-70B-Instruct-Turbo`, T=0.0, max_tokens=1024. Out-of-family by design (none of the 5 benchmarked models). Env var: `TOGETHER_API_KEY` (per [.env.example](.env.example), declared 2026-05-10). Cost ≈ $8 for ~3,800 judge calls at $0.88/M tokens.

Outputs: [data/processed_data/results_v2/llm_judge_scores_full.jsonl](data/processed_data/results_v2/llm_judge_scores_full.jsonl) (base runs), `perturbation_judge_scores.jsonl` (perturbation runs), `error_taxonomy_v2_judge.jsonl` (failure classification).

### Aggregation locus

Single-path aggregation: each task contributes one normalized score, models are averaged across all tasks they completed. See `llm-stats-vault/90-archive/audit/aggregation_locus.md` for the rationale (avoids the two-stage averaging bias that earlier v1 analyses exhibited).

## 6. Three-ranking framework — headline finding

The same five models reorder when scored on different axes. This is the central empirical claim.

### Ranking by accuracy (Path A normalized score, [data/processed_data/results_v1/results.json](data/processed_data/results_v1/results.json) `model_aggregates`)

1. **Claude** 0.683
2. Mistral 0.644
3. Gemini 0.642
4. DeepSeek 0.625
5. ChatGPT 0.621

### Ranking by robustness (1 − Δ between base and perturbation pass rate, [data/processed_data/results_v2/robustness_v2.json](data/processed_data/results_v2/robustness_v2.json))

1. **ChatGPT** 0.9996 (Δ = 0.0003)
2. Mistral 0.9981 (Δ = 0.0013)
3. Gemini 0.9824 (Δ = 0.0129)
4. Claude 0.9565 (Δ = 0.0305)
5. DeepSeek 0.9422 (Δ = 0.0388)

### Ranking by calibration (binary-pass ECE, lower = better, [data/processed_data/results_v2/calibration.json](data/processed_data/results_v2/calibration.json))

1. **Claude** 0.0334
2. ChatGPT 0.0339
3. Gemini 0.0765
4. Mistral 0.0811
5. DeepSeek 0.1977

> **Note on the "top-2 swap" headline.** The poster narrative ([poster/main.tex](poster/main.tex)) frames the headline as Claude leads accuracy / ChatGPT leads calibration. Binary-pass ECE places Claude marginally ahead on calibration (0.0334 vs 0.0339, a 0.0005 gap that is within bootstrap CI). The clearer divergence is **Claude 1st on accuracy and calibration → 4th on robustness**, while **ChatGPT 5th on accuracy → 1st on robustness**. The per-dimension calibration view ([per_dim_calibration.json](data/processed_data/results_v2/per_dim_calibration.json)) shows ChatGPT leads on the R (reasoning) dimension specifically. ### Supporting findings

- **Assumption-compliance gap.** L1 failure taxonomy ([data/processed_data/results_v2/error_taxonomy_v2.json](data/processed_data/results_v2/error_taxonomy_v2.json), n=143 judge-classified failures): `ASSUMPTION_VIOLATION` 67, `MATHEMATICAL_ERROR` 48, `FORMATTING_FAILURE` 18, `CONCEPTUAL_ERROR` 10. Assumption violation is the dominant failure mode (47%).
- **Keyword vs judge agreement** ([keyword_vs_judge_agreement.json](data/processed_data/results_v2/keyword_vs_judge_agreement.json), n=750 paired runs):
  - `assumption_compliance`: Spearman ρ = 0.602
  - `reasoning_quality`: Spearman ρ = 0.180
  - `method_structure`: Spearman ρ = 0.004
  Krippendorff α matches: assumption-compliance α = 0.573 (questionable but workable), other two near zero.
- **Pass-rate divergence on assumptions**: keyword rubric passes 71.1% on the assumption_compliance dimension; judge passes 55.7%. Keyword rubric is systematically more lenient on assumption checks (see [combined_pass_flip_analysis.json](data/processed_data/results_v2/combined_pass_flip_analysis.json) Phase 1.5 audit).

## 8. Reproducibility entry points

```bash
# Set up
source .venv/bin/activate
cp .env.example .env  # then fill 5 model keys + TOGETHER_API_KEY

# Regenerate tasks (171)
python -m baseline.bayesian.build_tasks_bayesian        # → tasks.json (136)
python -m baseline.bayesian.build_tasks_advanced        # → tasks_advanced.json (35)

# Regenerate perturbations (473)
python code/scripts/generate_perturbations_full.py           # → perturbations_v2.json (398)
                                                        # merged with 75 v1 → perturbations_all.json

# Run benchmark on all 5 models
python -m llm_runner.run_all_tasks --models claude gemini chatgpt deepseek mistral

# Run perturbation benchmark
python code/scripts/score_perturbations.py

# Refresh full analysis pipeline (post-runs)
bash code/scripts/refresh_pipeline.sh                        # all steps
bash code/scripts/refresh_pipeline.sh --no-judge             # skip the Llama judge

# Run external Llama judge separately (if --no-judge was used)
python -m evaluation.llm_judge_rubric \
    --runs data/processed_data/results_v1/runs.jsonl \
    --tasks data/raw_data/benchmark_v1/tasks_all.json \
    --output data/processed_data/results_v2/llm_judge_scores_full.jsonl

# Regenerate figures (R pipeline)
cd code/visualization/ && Rscript run_all.R

# Tests
pytest code/data_preprocessing/frequentist/test_frequentist.py code/capstone_mcp/test_server.py -v   # 53 tests

# Lint
ruff check .
```
