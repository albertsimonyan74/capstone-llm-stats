# Tier 2A.6 — pre-swap snapshot (2026-05-05)

This directory holds the visualization assets that were retired by **Tier
2A.6**. Files are preserved for git-free rollback and to satisfy any external
references (poster drafts, archived blog posts) that still link to the old
filenames.

## What changed

The Tier 2A.6 redesign:

1. **Replaced** `three_rankings.png` + `rank_shift.png` with a single
   `dimension_leaderboard.png` (three horizontal-bar panels, each sorted by
   its own metric, sharing model identity colors). The new chart makes the
   rank-inversion story visible in a single glance instead of relying on a
   bump-chart crossing pattern.
2. **Reoriented** `robustness_heatmap.png` from horizontal (5 rows × N cols)
   to vertical (N rows × 5 cols) with category brackets on the LEFT and
   every non-NaN cell annotated. The previous `robustness_heatmap_full.png`
   was a portrait sibling of the same data — it is now redundant.
3. **Promoted** `calibration_ece_ranking.png` to the §5 calibration primary;
   the single-panel `calibration_reliability.png` was already orphaned in
   the manifest (the small-multiples version is canonical) and is now
   archived. The poster-export copy in `report_materials/figures/` is still
   regenerated by `scripts/calibration_analysis.py`.

## Restore

If a Tier 2A.6 swap needs to be reverted for a single chart:

```bash
# Example: restore three_rankings to both locations
cp archive/visualizations-pre-tier2a6/three_rankings.report_materials.png \
   report_materials/figures/three_rankings.png
cp archive/visualizations-pre-tier2a6/three_rankings.public_v2.png \
   capstone-website/frontend/public/visualizations/png/v2/three_rankings.png
```

Then add the corresponding manifest entry back to
`capstone-website/frontend/src/data/visualizations.js` (refer to the git
history of that file for the verbatim entry shape).
