\section{Methodology}
\label{sec:methodology}

\subsection{Task Suite}

171 base tasks across 38 task types in two phases. Phase~1 (136
tasks, 31 types) covers closed-form conjugate Bayesian models:
Beta-Binomial, Gamma-Poisson, Normal mean with known and unknown
variance (Normal-Inverse-Gamma), Dirichlet-Multinomial, Bayesian
linear regression, and decision theory under squared, absolute, and
0-1 loss. Phase~2 (35 tasks, 7 types, 5 tasks each) covers
computational methods: Gibbs sampling, Metropolis-Hastings,
Hamiltonian Monte Carlo, Reversible-Jump MCMC, hierarchical models,
Variational Bayes, and Approximate Bayesian Computation. Every task
has a ground-truth solver implemented from
textbooks~\cite{bolstad2007bayesian,hoff2009bayesian,bishop2006prml}.
Phase~2 stochastic methods use seeded Monte Carlo estimates as
ground truth, with tolerances of 0.05 for MCMC variants and 0.10
for VB and ABC.

\subsection{Perturbation Taxonomy}

Figure~\ref{fig:pipeline} summarizes the full evaluation flow:
task generation feeds into perturbation generation, parallel model
querying, and three-path scoring. Each base task is perturbed along
three axes. \emph{Rephrase}
preserves the math and answer while rewording the prompt.
\emph{Semantic} preserves the math while relocating the problem to
a different real-world framing. \emph{Numerical} changes the inputs
and recomputes ground truth. The full set is 473 records: 75
hand-authored seeds covering 25 base tasks on all three axes, and
398 LLM-generated perturbations covering the remaining 146 tasks,
produced by the same Llama 3.3 70B Turbo used as the judge. Every
generated perturbation was checked against the ground-truth solver
before inclusion. Mean coverage is 2.77 per base task. The numerical
axis is skipped for 40 conceptual or seeded-MC tasks where swapping
numbers would invalidate the ground-truth procedure.

\subsection{Three-Path Scoring}

\textbf{Path~A} is a deterministic keyword rubric in the live
runner: extract the answer line, compare numeric targets to ground
truth within a per-task tolerance, check required structural and
assumption keywords in the prose. \textbf{Path~B} is a post-hoc
NMACR aggregator. Each task contributes five components in $[0,1]$:
numerical accuracy (N), method structure (M), assumption compliance
(A), confidence calibration (C), and reasoning quality (R). The
combined score is
\begin{equation}
  S_{\text{NMACR}} = 0.10\,N + 0.20\,M + 0.30\,A + 0.15\,C + 0.25\,R,
  \label{eq:nmacr}
\end{equation}
with weights drawn from the chain-of-thought, calibration, and
assumption-articulation
literature~\cite{wei2022cot,chen2022pot,guo2017calibration,du2025icecream,park2025judge,nagarkar2026canllm,fermieval2025,liu2025matheval}.
A is weighted highest because assumption articulation is what
Bayesian reasoning is for; N lowest because a correct number with
no derivation does not answer the question. Pass threshold is
$s \geq 0.5$. \textbf{Path~C}, the external judge, runs on one
dimension. Each (task, response) pair goes to Llama 3.3 70B Turbo
via Together AI with a fixed rubric prompt asking whether the
response explicitly states the assumptions the task requires. The
narrow scope addresses the holistic-judge failure modes Feuer et
al.~\cite{judgment2025noise} document.

\subsection{Three-Ranking Framework}

Three rank orders come from the same response corpus. The
\emph{accuracy ranking} uses Path~B NMACR pass rate per model on
the 171 base tasks. The \emph{robustness ranking} uses the
perturbation pass-flip rate. Let
$\mathrm{pass}(t, m) \in \{0,1\}$ be the base pass for task $t$ and
model $m$, and $\overline{\mathrm{pass}}(t, m)$ the mean pass over
$t$'s perturbations $\mathcal{P}(t)$. Then
\begin{equation}
  R_m = 1 - \frac{1}{|\mathcal{T}|}\sum_{t \in \mathcal{T}}
        \left| \mathrm{pass}(t, m) - \overline{\mathrm{pass}}(t, m) \right|,
  \label{eq:robustness}
\end{equation}
with $R_m = 1$ for perfect base/perturbation agreement. The
\emph{calibration ranking} uses binary-pass Expected Calibration
Error~\cite{guo2017calibration} over $B$ confidence buckets,
\begin{equation}
  \mathrm{ECE} = \sum_{b=1}^{B} \frac{|B_b|}{N}
                 \,\bigl|\,\mathrm{acc}(B_b) - \mathrm{conf}(B_b)\,\bigr|,
  \label{eq:ece}
\end{equation}
with $B = 4$ buckets (low, medium, high, unstated) parsed from the
response's hedging language. The three rankings are not redundant:
a model can be accurate but brittle, accurate and well-calibrated
but brittle, or robust but inaccurate. \S\ref{sec:results} shows
all three configurations occur in our five-model sample.

\subsection{Experimental Setup}

The five LLMs are queried directly over their HTTP APIs with no
vendor SDKs: Claude Sonnet 4.5 (Anthropic Messages), GPT-4.1 (OpenAI
Chat Completions), Gemini 2.5 Flash (Google Generative Language),
DeepSeek-Chat, Mistral Large. Decoding is identical across providers:
temperature at the API default (around 1.0), 1024-token max output,
60-second timeout, retries on transient 429 and 5xx, the same
Bayesian-statistics system prompt for every model. Each (task,
model) pair produces one response; a 3-sample self-consistency
subset is used for the calibration analysis. Bootstrap CIs use
10{,}000 non-parametric resamples (seed 42) of per-run aggregate
scores. A single shell script drives deduplication, scoring,
perturbation scoring, judge invocation, and aggregation; 53 unit
tests cover the task generators and the MCP query interface.
Figures come from the R pipeline at
\emph{report\_materials/r\_analysis/run\_all.R}.
