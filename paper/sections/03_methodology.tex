\section{Methodology}
\label{sec:methodology}

\subsection{Task Suite}

The benchmark consists of 171 base tasks across 38 task types,
partitioned into two phases. Phase~1 (136 tasks, 31 types) covers
closed-form conjugate Bayesian models: Beta--Binomial, Gamma--Poisson,
Normal mean with known and unknown variance (Normal--Inverse-Gamma),
Dirichlet--Multinomial, Bayesian linear regression, and decision-theory
tasks (Bayes risk under squared, absolute, and 0--1 loss). Phase~2
(35 tasks, 7 types, 5 tasks each) covers computational Bayesian
methods: Gibbs sampling, Metropolis--Hastings, Hamiltonian Monte
Carlo, Reversible-Jump MCMC, hierarchical models, Variational Bayes,
and Approximate Bayesian Computation. Each task includes a
ground-truth solver implemented from textbooks~\cite{bolstad2007bayesian,hoff2009bayesian,bishop2006prml};
stochastic methods (Phase~2) use seeded MC estimates as ground truth
with tolerances of 0.05 for MCMC variants and 0.10 for VB and ABC.

\subsection{Perturbation Taxonomy}

Each base task can be perturbed along three principled axes.
\emph{Rephrase} preserves the mathematics and the answer while
rewording the prompt; \emph{semantic} preserves the mathematics while
relocating the problem to a different real-world framing; and
\emph{numerical} changes the numerical inputs and recomputes the
ground-truth answer. The full perturbation set contains 473 records:
75 hand-authored seeds (covering 25 base tasks across all three axes)
and 398 LLM-generated perturbations (covering the remaining 146 base
tasks), produced by the same external Llama 3.3 70B Turbo used as the
assumption-compliance judge. All LLM-generated perturbations were
filtered against the ground-truth solver before inclusion, which acts
as an automated validity check. Mean coverage is 2.77 perturbations
per base task; the numerical axis is skipped for 40 conceptual or
seeded-MC tasks where number replacement would invalidate the
ground-truth procedure.

\subsection{Three-Path Scoring}

Each model response is scored by three independent paths.
\textbf{Path~A} (a keyword rubric in the live runner) is a
deterministic keyword rubric: it extracts the answer line, compares
numeric targets against the ground truth within a per-task tolerance,
and checks for required structural and assumption keywords in the
free-form prose. \textbf{Path~B} (a post-hoc NMACR aggregator) is the
rubric-based NMACR aggregator. Each task contributes five components
in $[0,1]$: numerical accuracy (N), method structure (M), assumption
compliance (A), confidence calibration (C), and reasoning quality (R).
The combined score is a literature-weighted sum
\begin{equation}
  S_{\text{NMACR}} = 0.10\,N + 0.20\,M + 0.30\,A + 0.15\,C + 0.25\,R
  \label{eq:nmacr}
\end{equation}
with weights derived from the chain-of-thought, calibration, and
assumption literatures~\cite{wei2022cot,chen2022pot,guo2017calibration,du2025icecream,park2025judge,nagarkar2026canllm,fermieval2025,liu2025matheval}.
The largest weight (A) reflects the centrality of assumption
articulation in Bayesian reasoning; the smallest (N) reflects that a
correct number with no derivation is not the goal. Pass threshold is
$s \geq 0.5$. \textbf{Path~C}, the external judge, is constrained to
one dimension. Each (task, response) pair is sent to Llama 3.3 70B
Turbo via the Together AI API with a fixed rubric prompt asking
whether the response explicitly states the assumptions the task
requires. The constrained scope is a deliberate response to the
holistic-judge failure modes documented by Feuer et
al.~\cite{judgment2025noise}.

\subsection{Three-Ranking Framework}

Three rank orders are computed from the same response corpus. The
\emph{accuracy ranking} uses the Path~B NMACR pass rate per model
across all 171 base tasks. The \emph{robustness ranking} uses the
perturbation pass-flip rate. For each base task $t$ and model $m$,
let $\mathrm{pass}(t, m) \in \{0,1\}$ be the binary base pass and
$\overline{\mathrm{pass}}(t, m)$ the mean pass over $t$'s perturbations
$\mathcal{P}(t)$. The model's robustness is
\begin{equation}
  R_m = 1 - \frac{1}{|\mathcal{T}|}\sum_{t \in \mathcal{T}}
        \left| \mathrm{pass}(t, m) - \overline{\mathrm{pass}}(t, m) \right|,
  \label{eq:robustness}
\end{equation}
with $R_m = 1$ indicating perfect base/perturbation agreement.
Bootstrap CIs use 10{,}000 non-parametric resamples (seed 42). The
\emph{calibration ranking} uses binary-pass Expected Calibration
Error~\cite{guo2017calibration} over $B$ confidence buckets:
\begin{equation}
  \mathrm{ECE} = \sum_{b=1}^{B} \frac{|B_b|}{N}
                 \,\bigl|\,\mathrm{acc}(B_b) - \mathrm{conf}(B_b)\,\bigr|,
  \label{eq:ece}
\end{equation}
where $B_b$ is the set of predictions in bucket $b$,
$\mathrm{acc}(B_b)$ is empirical accuracy within the bucket, and
$\mathrm{conf}(B_b)$ is the bucket's mean confidence. We use $B = 4$
buckets (low, medium, high, unstated) parsed from the response's
hedging language. These three
rankings are not redundant: a model can be accurate but brittle
(high accuracy, low robustness), accurate and well-calibrated but
still brittle, or robust but low-accuracy. The central empirical
claim of this paper (\S\ref{sec:results}) is that all three of those
configurations are realised among the five models we benchmark.

\subsection{Experimental Setup}

\textbf{Models and APIs.} We benchmark five frontier LLMs queried
directly over their HTTP APIs, with no vendor SDK abstractions: Claude
Sonnet 4.5 (Anthropic Messages API), GPT-4.1 (OpenAI Chat Completions),
Gemini 2.5 Flash (Google Generative Language API), DeepSeek-Chat
(DeepSeek), and Mistral Large (Mistral). Common decoding settings are
shared across providers: temperature is left at the API default
(typically near 1.0 for the chat endpoints), maximum output is 1024
tokens, and each call carries a 60-second timeout with retry on
transient 429 and 5xx responses. The same Bayesian-statistics system
prompt is used for every model.

\textbf{Evaluation pipeline.} Each (task, model) pair produces one
response. Path~A extracts the final answer line and structural keywords;
Path~B aggregates the five NMACR components into a weighted score; and
the external Llama 3.3 70B Turbo judge, served by Together AI's
OpenAI-compatible API, scores the response on assumption-compliance
alone. Perturbations are evaluated with the same prompts and the same
three paths. Single-pass per (task, model) is the default; a small
self-consistency subset (3 samples) is used for the calibration
analysis.

\textbf{Calibration computation.} Equation~\ref{eq:ece} is computed
per model on the full response corpus. Uncertainty is reported as
95\% percentile bootstrap confidence intervals from 10{,}000
non-parametric resamples of per-run aggregate scores (seed 42), the
same procedure used for the accuracy and robustness CIs.

\textbf{Reproducibility.} The full analysis pipeline is driven by a
single shell script that runs deduplication, scoring, perturbation
scoring, judge invocation, and aggregation in order. Unit tests
(53 in total) cover the task generators and the MCP query interface.
All figures in this paper are generated programmatically by the R
analysis pipeline at \emph{report\_materials/r\_analysis/run\_all.R};
no figure is hand-edited.
