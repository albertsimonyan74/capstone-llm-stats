## Conventions

- All figures use `apply_print_theme()` from `print_theme.py`
- Model colors via `MODEL_COLORS_PRINT` (darker -600 tailwind shades)
- Save via `dual_save(fig, basename)` to `poster/figures/` — emits SVG + PNG only, no PDF
- Fonts kept as text in SVG (`svg.fonttype: none`) — selectable, lossless
- Figsize chosen for poster column width: ~14 inches for full-width rows

## Rendering `poster.html` to print-ready PDF

`poster/poster.html` is a single self-contained file (no external image refs, fonts loaded from Google Fonts CDN at print time). All 7 SVGs are inlined as `<svg>` blocks so vector fidelity and selectable text survive the PDF export.

1. Open `poster/poster.html` in Google Chrome (Chromium also works).
2. Wait until Google Fonts (Space Grotesk, Space Mono) finish loading — give it ~2 s after first paint.
3. File → Print (or ⌘P / Ctrl+P).
4. Destination: **Save as PDF**.
5. Paper size: **Custom · 841 mm × 1189 mm** (A0 portrait). If "Custom" is unavailable, choose **A0** if listed.
6. Margins: **None**.
7. Layout: **Portrait**.
8. Background graphics: **on** (else `print-color-adjust:exact` panels lose their fills).
9. Save → take resulting PDF to print shop. Confirm at the shop that it renders at 100% scale on A0 stock.

### Regenerating

The HTML is generated by `poster/scripts/build_poster_html.py`. Re-run after any figure regeneration:

```bash
python poster/scripts/build_poster_html.py
```

The script reads each SVG from `poster/figures/`, strips the `<?xml?>`/`<!DOCTYPE>` preamble, force-sizes the root `<svg>` to `width:100%; height:auto`, and inlines every figure into `poster/poster.html`. Headline numbers in the judge-validation caption are pulled from `poster/figures/derived/disagreement_matrix_2x2.json` (`n_total = 2,850`, `kw_pass_judge_fail = 591`, `directional_pass_flip_pct ≈ 20.74`).
