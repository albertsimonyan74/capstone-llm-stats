\section{Methodology}
\label{sec:methodology}

\subsection{Task Suite}

Two phases are passed for 171 base tasks across 38 task types.
Phase~1 (136 tasks, 31 types) covers closed-form conjugate Bayesian
models: Beta-Binomial, Gamma-Poisson, Normal mean with known and
unknown variance (Normal-Inverse-Gamma), Dirichlet-Multinomial,
Bayesian linear regression, and decision theory under squared,
absolute, and 0-1 loss. Phase~2 (35 tasks, 7 types, 5 tasks each)
covers computational methods: Gibbs sampling, Metropolis-Hastings,
Hamiltonian Monte Carlo, Reversible-Jump MCMC, hierarchical models,
Variational Bayes, and Approximate Bayesian Computation. Based on
the implementation from
textbooks~\cite{bolstad2007bayesian,hoff2009bayesian,bishop2006prml},
each task has a ground-truth solver. Seeded Monte Carlo estimates
are used as ground truth for Phase~2 stochastic methods, with
tolerances of 0.05 for MCMC variants and 0.10 for VB and ABC.

\subsection{Perturbation Taxonomy}

The full evaluation flow is summarized in Figure~\ref{fig:pipeline}
below. Task generation feeds into perturbation generation, parallel
model querying, and three-path scoring. Perturbation for each task
is done in three axes. \emph{Rephrasing} keeps the math and answer
while rewording the prompt. \emph{Semantic} perturbation keeps the
math, but relocates the problem context to a different real-world
framing. \emph{Numerical} changes the inputs and recomputes the
ground-truth. The full perturbation set is 473 records, 75 of which
are hand-written seeds covering 25 base tasks on all three axes.
The remaining 398 are LLM-generated perturbations (produced by the
LLM judge Llama 3.3 70B Turbo) covering the remaining 146 tasks.
Ground-truth solver was checked against each generated perturbation
before inclusion. For 40 conceptual or seeded-MC tasks, the
numerical axis is skipped because number swapping would cause an
invalid ground-truth procedure.

\subsection{Three-Path Scoring}

\textbf{Path~A} --- the deterministic keyword rubric extracts the
answer line, compares numeric targets to ground-truth within a
per-task tolerance, and checks required structural and assumption
articulation related keywords in the prose. \textbf{Path~B} --- a
post-hoc NMACR aggregator. Each task contributes five components
from five corresponding domains in the range of $[0,1]$ ---
Numerical Accuracy (N), Method Structure (M), Assumption Compliance
(A), Confidence Calibration (C), and Reasoning Quality (R). The
combined score is
\begin{equation}
  S_{\text{NMACR}} = 0.10\,N + 0.20\,M + 0.30\,A + 0.15\,C + 0.25\,R,
  \label{eq:nmacr}
\end{equation}
with weights drawn from the chain-of-thought, calibration, and
assumption-articulation
literature~\cite{wei2022cot,chen2022pot,guo2017calibration,du2025icecream,park2025judge,nagarkar2026canllm,fermieval2025,liu2025matheval}.
The highest weight in the formula is allocated to A because
assumption articulation is the foundation of Bayesian reasoning.
N is given the lowest weight since correct numeric calculation
without derivation does not support the answer logic of the
specified question. The pass threshold is set to $s \geq 0.5$.
\textbf{Path~C} --- the external judge runs on one dimension.
Llama 3.3 70B Turbo via Together AI is fed with each (task,
response) pair and a fixed prompt rubric, which requires a statement
from the judge whether the explicit assumption articulation
requirement is met. The holistic-judge failure modes are addressed
in a narrower scope in the Feuer et al.~\cite{judgment2025noise}
document.

\subsection{Three-Ranking Framework}

Three rank orders come from the same response corpus. The
\emph{accuracy ranking} uses the mean Path~B NMACR score per model
on the 171 base tasks. The \emph{robustness ranking} uses the
ratio of mean NMACR score on perturbation runs to mean NMACR score
on the paired base runs over the $n = 398$ paired records. With
$\bar{s}_{\text{base}}(m)$ and $\bar{s}_{\text{pert}}(m)$ those
means,
\begin{equation}
  R_m = \frac{\bar{s}_{\text{pert}}(m)}{\bar{s}_{\text{base}}(m)}
      = 1 - \frac{\Delta_m}{\bar{s}_{\text{base}}(m)},
  \label{eq:robustness}
\end{equation}
where $\Delta_m = \bar{s}_{\text{base}}(m) - \bar{s}_{\text{pert}}(m)$
is the signed mean score gap; $R_m = 1$ is perfect agreement. The
\emph{calibration ranking} uses binary-pass Expected Calibration
Error~\cite{guo2017calibration} over $B$ confidence buckets,
\begin{equation}
  \mathrm{ECE} = \sum_{b=1}^{B} \frac{|B_b|}{N}
                 \,\bigl|\,\mathrm{acc}(B_b) - \mathrm{conf}(B_b)\,\bigr|,
  \label{eq:ece}
\end{equation}
with $B = 4$ buckets (low, medium, high, unstated) parsed from the
response's hedging language. The three rankings are not redundant:
a model can be accurate but brittle, accurate and well-calibrated
but brittle, or robust but inaccurate. \S\ref{sec:results} shows
all three configurations occur in the five-model sample.

\subsection{Experimental Setup}

The five LLMs are queried directly over their HTTP APIs with no
vendor SDKs: Claude Sonnet 4.5 (Anthropic Messages), GPT-4.1 (OpenAI
Chat Completions), Gemini 2.5 Flash (Google Generative Language),
DeepSeek-Chat, and Mistral Large. The benchmark run sends no
explicit temperature, so each provider's API default applies (1.0
for Anthropic, OpenAI, Google, and DeepSeek, and 0.7 for Mistral). Max
output is 1024 tokens for Claude, GPT-4.1, DeepSeek, and Mistral,
and 4096 tokens for Gemini (its thinking-token accounting consumes
part of the budget before emitting visible text). Other decoding
parameters are shared: a 60-second per-request timeout, retries on
transient 429 and 5xx responses, and the same Bayesian-statistics
system prompt for every model. Each (task, model) pair produces one
response. The calibration analysis uses a separate 3-rerun
self-consistency protocol at temperature $T = 0.7$ with
top-$p = 0.95$.
Bootstrap CIs use 10{,}000 non-parametric resamples (seed 42) of
per-run aggregate scores. A single shell script drives deduplication,
scoring, perturbation scoring, judge invocation, and aggregation.
53 unit tests cover the frequentist statistical solvers and the MCP
query interface.
